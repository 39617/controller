1 MC1322x
2 =======
3 
4 libmc1322x is a library, build system, test code, and utilities for using the
5 mc13224v from Freescale.

7 Getting Started
8 ---------------
9 
10  cd tests
11  make
12 
13 this will build all the test files in libmc1322x/tests for each board
14 defined in libmc1322x/board. You will have programs like:
15 
16 rftest-tx_redbee-dev.bin
17 rftest-tx_redbee-r1.bin
18 
19 rftest-rx_redbee-dev.bin
20 rftest-rx_redbee-r1.bin
21 
22 if you only wanted to build binaries for one board you can do:
23 
24  make BOARD=redbee-dev
25 
26 You can use mc1322x-load.pl in libmc1322x/tools to run your code:
27 
28  ../tools/mc1322x-load.pl -f rftest-tx_redbee-dev.bin

30 Incorporating libmc1322x into your own code
31 -------------------------------------------
32 
33 The best way to incorporate libmc1322x into your code is as a git submodule:
34 
35  mkdir newproject
36  cd newproject
37  git init
38 
39 Initialized empty Git repository in /home/malvira/newproject/.git/
40 
41  git submodule add git://git.devl.org/git/malvira/libmc1322x.git
42 
43 This will add libmc1322x to your repository. Now to setup the Makefile:
44 
45  cp libmc1322x/tests/Makefile .
46 
47 You need to edit the Makefile to point MC1322X to your libmc1322x submodule:
48 
49 Change line 1
50 
51  MC1322X := ..
52 
53 to
54 
55  MC1322X := libmc1322x
56 
57 and edit COBJS and TARGETS accordings. COBJS are all of your common code for
58 any of your programs. TARGETS are the names of your programs.
59 
60 For instance, you can have a common routine that prints a welcome message that
61 is used by two programs a and b. You would add common.o to COBJS:
62 
63  COBJS:= common.o
64 
65 and your target line would read:
66 
67  TARGETS := a b
68 
69 COBJS are made for each board so it is ok to have board specific code in there.
70 As an example, tests uses this in tests.c to print the name of the board in the
71 welcome message. You could also use this to change your GPIO mappings between
72 boards.
