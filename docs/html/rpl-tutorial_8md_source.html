1 Contiki RPL Quickstart using Econotags
2 ======================================
3 
4 Contiki RPL implements the IETF ROLL working group's RPL: IPv6 Routing
5 Protocol for Low power and Lossy Networks. This routing standard
6 enables you to set up a route-over 6LowPAN network.
7 
8 This tutorial will show you how to run Contiki's RPL demos on Redwire
9 Econotags.
10 
11 1) Obtain Redwire Econotags, or other mc1322x hardware.
12 ---------------------------------------------------------------
13 
14 See [the mc1322x hardware](http://mc1322x.devl.org/hardware.html) for
15 an incomplete, but growing list of mc1322x hardware.
16 
17 We recommend a few [Econotags](http://www.redwirellc.com/store/node/1)
18 to get started. In addition to the mc13224v, the Econotag also has the
19 FT2232H USB dual port controller. One port is connected to UART1 and
20 all the necessary pins to load, erase, and reset the mc13224v. The
21 other port is connected to the JTAG pins for run-time debugging. **No
22 other programming hardware is necessary. Just your computer!**
23 
24 2) Get a toolchain, and Contiki source.
25 ----------------------------------------
26 
27 Everything you need is in [Instant Contiki
28 2.5](http://www.sics.se/contiki/news/contiki-2.5-release-candidate-1-avaliable.html). If
29 you are going to be working primarily with Contiki then this is the
30 way to go.
31 
32 Instant Contiki contains the
33 [mc1322x-oe](http://git.devl.org/?p=malvira/mc1322x-oe.git;a=summary)
34 [[ubuntu]](http://mc1322x.devl.org/ubuntu.html)
35 [[mac]](http://mc1322x.devl.org/mac.html) toolchain.
36 
37 Development of the mc1322x port occurs in a different branch from the
38 main Contiki CVS repository. The most current mc1322x contiki port can be found here:
39 [contik-mc1322x.git](http://git.devl.org/?p=malvira/contiki-mc1322x.git)
40 
41 3) Build the border-router and rpl-udp
42 ---------------------------------------
43 
44 The RPL demo consists of a "Border router" and at least one other RPL
45 enabled node. For this tutorial, the border router creates a RPL DAG
46 and is the ROOT. The other node will be a RPL udp-client. It joins the
47 RPL network, acts as an RPL router, and sends a UDP datagram to a
48 hardcoded IPV6 address. This tutorial only handles a two node network,
49 but can be extended for any number of nodes.
50 
51 The general command to make a Contiki binary for the Econotag is:
52 
53  make TARGET=redbee-econotag
54 
55 To build the border-router:
56 
57  cd contiki-2.x/examples/ipv6/rpl-border-router
58  make TARGET=redbee-econotag
59 
60 This will produce the binary image
61 'border-router_redbee-econotag.bin', which you can load directly on to
62 an mc1322x and execute.
63 
64 To build the udp-client, do:
65 
66  cd contiki-2.x/examples/ipv6/udp-client
67  make TARGET=redbee-econotag
68 
69 which will build 'udp-client_redbee-econotag.bin'.
70 
71 4) Load the binaries on the Econotags:
72 -------------------------------------------
73 
74 ### a) Run the econotags with randomized MAC addresses (random, but persistent)
75 
76 After building border-router_redbee-econotag.bin, you load the code
77 with:
78 
79  mc1322x-load.pl -f border-router_redbee-econotag.bin -t /dev/ttyUSB1 -c 'bbmc -l redbee-econotag reset'
80 
81 The econotag will flash itself with a random Redwire MAC address if
82 you haven't programmed one in:
83 
84  address in flash blank, setting to defined IAB with a random extention.
85  flashing blank address
86  Rime started with address 00:50:C2:FF:FE:A8:C2:66
87  CSMA sicslowmac, channel check rate 100 Hz, radio channel 26
88  Tentative link-local IPv6 address
89  fe80:0000:0000:0000:0250:c2ff:fea8:c266
90  Tentative global IPv6 address aaaa:0000:0000:0000:0250:c2ff:fea8:c266
91 
92 This mac address is used to derive its IPv6 address. In this example,
93 it's aaaa::0250:c2ff:fea8:c266.
94 
95 The border router connects to your computer over a SLIP tunnel formed
96 by `tunslip6` --- a tool included in Contiki.
97 
98 To make tunslip6:
99 
100  cd contiki-2.x/tools
101  make tunslip6
102 
103 You need to run tunslip6 to create this tunnel and SLIP connection:
104 
105  tunslip6 -s /dev/ttyUSB1 aaaa::1/64
106 
107 ### b) Program the MAC addresses on the Econotag
108 
109 To erase the econotag (and the MAC address), first make sure you've
110 built BBMC:
111 
112  cd contiki-2.x/cpu/mc1322x/tool/ftditools
113  make
114 
115 If you have trouble, see the [BBMC
116 page](http://mc1322x.devl.org/bbmc.html) for details. BBMC controls
117 the FT2232H over USB through libftdi. It can reset or erase an
118 mc13224v. To erase:
119 
120  bbmc -l redbee-econotag erase
121 
122 Then to flash just the MAC address you need to build flasher from
123 libmc1322x. See:
124 
125  http://mc1322x.devl.org/libmc1322x.html
126 
127 Then to flash the MAC address you can do:
128 
129  mc1322x-load.pl -e -f flasher_redbee-econotag.bin -z -t /dev/ttyUSB1 -c 'bbmc -l redbee-econotag reset' 0x1e000,0xABC25000,0x010000C0
130 
131 + -z to indicate you are sending a zero length binary to flasher.
132 + -e causes mc1322x-load.pl to exit when it's done, instead of
133  dropping to a terminal
134 
135 The next time you run Contiki you'll get:
136 
137  loading rime address from flash.
138  Rime started with address 00:50:C2:AB:C0:00:00:01
139  CSMA sicslowmac, channel check rate 100 Hz, radio channel 26
140  Tentative link-local IPv6 address
141  fe80:0000:0000:0000:0250:c2ab:c000:0001
142  Tentative global IPv6 address aaaa:0000:0000:0000:0250:c2ab:c000:0001
143 
144 N.B. for all of the bbmc commands, you'll need to use -i if you have
145 more than one econotag plugged in. On most systems the indexes are
146 enumerated backwards from the terminals. E.g. if you have 23 nodes
147 connected, then /dev/ttyUSB1 will be on ftdi index 22 and the load
148 command would be:
149 
150  mc1322x-load.pl -f foo.bin -t /dev/ttyUSB1 -c 'bbmc -l redbee-econotag -i 22 reset'
151 
152 5) Start the udp-client
153 -----------------------
154 
155 Use mc1322x-load.pl to run `udp-client_redbee-econotag.bin` on the
156 second econotag:
157 
158  mc1322x-load.pl -f udp-client_redbee-econotag.bin -t /dev/ttyUSB3
159  'bbmc -l redbee-econotag -i 0'
160 
161 6) Check the border router's webpage for status
162 ------------------------------------------------
163 
164 The RPL border-router serves a webpage that displays the known
165 neighbors and the known routes to all the node on the DAG. In firefox,
166 you can type in the following to go to a IPv6 address:
167 
168  http://[aaaa:0000:0000:0000:0250:c2ff:fea8:c266]
169 
170 #### Neighbors
171 fe80::250:c2ff:fea8:cbf4
172 #### Routes
173 aaaa::250:c2ff:fea8:cbf4/128 (via fe80::250:c2ff:fea8:cbf4)
174 
175 7) Ping6 all the nodes
176 ----------------------
177 
178 After a short time, once the DAG forms to a node, you should see a
179 route appear on the border-router's status page. You can ping this
180 node from the computer running 'tunslip6'
181 
182  ping6 aaaa::250:c2ff:fea8:cbf4
183 
184 A successful ping shows that both a downward and upward route has
185 formed.
186 
187 8) That's it!
