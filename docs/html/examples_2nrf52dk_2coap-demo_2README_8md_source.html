1 A CoAP demo for nRF52 DK
2 ========================
3 This demo contains two applications: coap-server and coap-client which are similar to
4 [Coap Observable Server] and [Coap Observer Client] examples provided by the nRF5 IoT SDK.
5 
6 Note that before any CoAP requests can be made you'll need to configure an IPv6 connection
7 to the device and assign a routable IPv6 address.
8 
9 For details how to do this please refer to sections 'Establishing an IPv6 connection'
10 and 'Distributing routable IPv6 prefix' in `platform/nrf52dk/README-BLE-6LoWPAN.md`.
11 
12 CoAP Server
13 ===========
14 The server exposes the following resources:
15 
16  host
17  |-- .well-known
18  | `-- core
19  `-- lights
20  `-- led3
21 
22 The state of LED 3 can be set and queried via CoAP through the observable resource `lights/led3`. Current
23 state of LED 3 is returned as a text string in the payload. The value 0 means that LED 3 is off, 1 otherwise.
24 
25 Button 1 can be used to toggle state of the LED 3. This will cause a notification to be sent to
26 any subscriber observing `lights/led3`. The state of the resource can also be changed by using POST/PUT to
27 the resource with desired state encoded as a text in the payload. Send 1 to switch on and 0 to switch off.
28 
29 In order to compile and flash the CoAP server to a DK execute:
30 
31  make TARGET=nrf52dk coap-server.flash
32 
33 Note, if you haven't previously used a given device with Contiki it is recommended
34 to erase the device and flash SoftDevice before flashing CoAP application, i.e.,
35 
36  make TARGET=nrf52dk erase
37  make TARGET=nrf52dk softdevice.flash
38 
39 Please refer to the *Testing* and *Python Example* sections of [Coap Observable Server] tutorial for detailed description how to query the Coap Server using a PC.
40 
41 CoAP Client
42 ===========
43 CoAP client compliments the CoAP server application. When Button 1 on the DK is pressed the the
44 client subscribes to `lights/led3` resource. If successful the LED 4 will blink briefly. From this moment
45 any change of the `lights/led3` resource will be automatically reflected by the client's LED 3.
46 
47 Note that the client must know the server's IPv6 address. The address is specified as a make variable
48 during compliation.
49 
50 In order to compile and flash the CoAP client to DK execute:
51 
52  make TARGET=nrf52dk SERVER_IPV6_ADDR=<full-ip6-address> coap-client.flash
53 
54 Note, that you can use `NRF52_JLINK_SN=<SN>` to select a particular devkit in a case when
55 you have more than one boards connected to PC. Please refer to `platform/README.md` for
56 details.
57 
58 Please refer to the *Testing* and *Python Server Example* sections of [Coap Observer Client] tutorial for detailed description how to use CoAP client demo with a PC.
