1 # HOWTO - Setting up the RPL Border router and other nodes
2 
3 In this folder we have a fully functional demonstrator with the components:
4 
5 1. **RPL border router**: to start the wireless network and connect it to other networks.
6 2. and a **wireless node** that acts as a basic RPL node by default (but can optionally be used configured as DAG Root)
7 
8 ## RICH RPL Border Router
9 
10 Setup the UART flow-control mode for the router from border-router/project-conf.h
11 
12 * Enable either **HW flow control**
13 ```C
14 #define UART_HW_FLOW_CTRL 1
15 #define UART_XONXOFF_FLOW_CTRL 0
16 ```
17 * or **SW flow control**
18 ```C
19 #define UART_HW_FLOW_CTRL 0
20 #define UART_XONXOFF_FLOW_CTRL 1
21 ```
22 * You can disable both, but it is not recommended.
23 
24 Compile and flash a node with the rpl-border-router.jn516x.bin image. Either a USB dongle or a dev-board would work.
25 
26 From a Linux terminal, go to `contiki/examples/jn516x/rpl/border-router` and do either
27 `make connect-router-hw` if you have **HW flow control**
28 or `make connect-router-sw` if you have **SW flow control**
29 
30 This will start a tunnel interface (tun0) on the host machine.
31 All traffic towards our network (prefix fd00::1/64) will now be routed to the border router.
32 
33 ## RPL Node
34 
35 The directory contiki-private/examples/jn516x/rpl/node contains a basic RICH node running TSCH and RPL.
36 You can compile and program more NXP nodes to run this, forming a larger network.
37 You should be able to ping the nodes (you can obtain their IPv6 address either directly from their log output
38 or by enabling DEBUG output in rpl-icmp6.c and looking at DAO prefixes being added.
