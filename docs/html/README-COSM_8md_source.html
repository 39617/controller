4 Short description on how to set-up a sensor network for global IPv6 addresses.
5 NOTE: this assumes that you do not have a native IPv6 connection.
6 
7 You will need:
8 
9 - PC with Ubuntu (Linux) - 11 or 12 versions
10 - A node for the RPL-Border-Router (examples/ipv6/rpl-border-router)
11 - A node for the json webservice (examples/ipv6/json-ws)
12 
13 Set-up IPv6 tunnel and Border Router
14 ------------------------------------
15 1. Ensure that you have gogo6c installed.
16 
17  sudo apt-get install gogoc
18 
19 2. Register an account at gogo6 and Freenet6 (http://www.gogo6.com).
20  The account at Freenet6 is needed by the gogo6c client.
21 
22 3. Edit the gogoc.conf and set your own Freenet6 user and password by
23  changing the lines with "userid" and "passwd".
24 
25 4. Start gogoc at command line
26 
27  cd contiki/examples/ipv6/json-ws
28  sudo gogoc -f gogoc.conf -n
29 
30 This will print your prefix - TSP_PREFIX. In my case
31 TSP_PREFIX=2001:05c0:1517:e400 (prefixlen is 56).
32 
33 5. Connect one of the nodes to the PC (via USB or serial) and program
34  it with the RPL-border-router (assumes Z1 node).
35 
36  cd contiki/examples/ipv6/rpl-border-router
37  make DEFINES=DEFINES=NETSTACK_RDC=nullrdc_driver,NULLRDC_CONF_802154_AUTOACK=1 TARGET=z1 border-router.upload
38 
39 6. Run tunslip6 which will forward IP from the RPL network to the IPv6 tunnel
40  (and to the Internet).
41 
42  cd contiki/examples/ipv6/rpl-border-router
43  make connect-router PREFIX=<TSP_PREFIX>::1/64
44 
45 When you start this you should get a printout from the border-router
46 which give you the IPv6 address of it.
47 
48 Server IPv6 addresses:
49  2001:05c0:1517:e400:c30c::10a
50  fe80::c30c:0:0:10a
51 
52 7. Browse using Mozilla Firefox (or any other browser) to the IPv6 address
53  given by the border router. This will show you the list of other nodes
54  connected to the RPL network.
55 
56  http://[2001:05c0:1517:e400:c30c::10a]/
57 
58  NOTE: this is a global IPv6 address so it should also be reachable from
59  any machine on the Internet.

61 Configuration of COSM submission
62 --------------------------------
63 1. Register a COSM account at https://cosm.com/
64  Set-up a feed and create an API key for the feed.
65 
66 2. Program the sensor node with (assumes Z1)
67 
68  cd contiki/examples/ipv6/json-ws
69  make websense-z1.upload WITH_COSM=1 TARGET=z1
70 
71 3. Check the IPv6 address of the node via the RPL-border router or by looking
72  at printouts when booting (make login TARGET=z1)
73 
74 4. You need to configure the node to push data to the COSM feed and this can be
75  done in several ways. For convenience a Python script is included that
76  pushes the configuration to the nodes.
77 
78  Edit the file 'setcosm.py' and replace "<your-key>" and "<your-feed>" with
79  your COSM API key and COSM feed id. You can then use this Python script to
80  configure your nodes.
81 
82  This is an example that configures the node with IP address
83  2001:05c0:1517:e400:c30c::10b to push data to the COSM feed with stream 1:
84 
85  cd contiki/examples/ipv6/json-ws
86  ./setcosm.py [2001:05c0:1517:e400:c30c::10b] 1
87 
88  Another way to configure the nodes is to use a REST add-on for the web
89  browser to post a COSM configuration to the node. "REST Client" for Mozilla
90  Firefox is an example of such add-on.
91 
92  POST a JSON expression to your node with the following data: This assumes
93  that you have the feed with id 55180 and want to post to stream 1 in that
94  feed. The field 'appdata' should be set to the API key you created at the
95  COSM web site for the feed.
96 
97  {
98  "host":"[2001:470:1f10:333::2]",
99  "port":80,
100  "path":"/v2/feeds/55180/datastreams/1",
101  "appdata":"<insert your COSM API key>",
102  "interval":120,
103  "proto":"cosm"
104  }
105 
106  This will configure the node to periodically push temperature data every
107  other minute. You can use GET to retrieve the data to se that the node has
108  been successfully configured (the COSM API key will be visualized as a
109  number of stars).
