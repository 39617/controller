1 This code connects a 802.15.4 radio over TTY with the full uIPv6 stack of
2 Contiki including 6LoWPAN and 802.15.4 framing / parsing. The native border
3 router also acts as a RPL Root and handles the routing and maintains the RPL
4 network. Finally the native border router connects the full 6LoWPAN/RPL
5 network to the host (linux/os-x) network stack making it possible for
6 applications on the host to transparently reach all the nodes in the
7 6LoWPAN/RPL network.
8 
9 This is designed to interact with the a ../slip-radio example running on a
10 mote that is either directly USB/TTY connected, or is remote via a TCP
11 connect. What's on the SLIP interface is really not Serial Line IP, but SLIP
12 framed 15.4 packets.
13 
14 The border router supports a number of commands on it's stdin.
15 Each are prefixed by !:
16 * !G - global RPL repair root.
17 * !M - set MAC address (if coming from RADIO, i.e. SLIP link)
18 * !C - show channel (if coming from RADIO, i.e. SLIP link)
19 * !D - sensor data received
20 * !Q - exit
21 
22 Queries are prefixed by ?:
23 * ?M is used for requesting the MAC address from the radio in order to use it for uIP6 and its stateless address auto configuration of its IPv6 address. This will make the native border router have the address that correspond to the MAC address of the slip-radio. (response is !M from the slip-radio)
24 
25 * ?C is used for requesting the currently used channel for the slip-radio. The response is !C with a channel number (from the slip-radio).
26 
27 * !C is used for setting the channel of the slip-radio (useful if the motes are using another channel than the one used in the slip-radio).
