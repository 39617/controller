1 MC1322x subtree
2 ===============
3 
4 The Contiki MC1322x port includes libmc1322x as a subtree. This makes pulling
5 updates to libmc1322x easy, but pushing changes from contiki to libmc1322x is
6 not so easy. However, this should not stop you from implementing core features
7 in contiki first, (especially if you are in a bind). The way to do this is to
8 make files prefixed with contiki- in cpu/mc1322x and add them to
9 Makefile.mc1322x.
10 
11 For instance, if you need a routine called sleep, but libmc1322x doesn't have
12 that yet, you could implement sleep in contiki-crm.c. Feel free to use as many
13 contiki specific things in here as you want. We can pull these changes directly
14 into the contiki tree. There is nothing stopping you from making changes to the
15 lib files --- in fact you should if that is the right thing to do (and then
16 push your changes upstream). The subtree merge should make it easy to still
17 pull updates.
18 
19 You'll also notice that the libmc1322x build system is still present. This
20 allows you to cd ./tests and make all of the libmc1322x unit tests as normal.
21 This is a handy way to perform a sainity check on all of the mc1322x specific
22 code.
23 
24 The subtree was set up as follows:
25 
26 From:
27 [http://www.kernel.org/pub/software/scm/git/docs/howto/using-merge-subtree.html](http://www.kernel.org/pub/software/scm/git/docs/howto/using-merge-subtree.html)
28 
29 Moved mc1322x.lds to mc1322x.lds.contiki
30 
31 Setup with:
32 
33  git remote add -f libmc1322x git://git.devl.org/git/malvira/libmc1322x.git
34  git read-tree --prefix=cpu/mc1322x -u libmc1322x/master
35  git commit -m "Merge libmc1322x as a subdirectory"
36 
37 And to do subsequent merges from libmc1322x do:
38 
39  git pull -s subtree libmc1322x master
