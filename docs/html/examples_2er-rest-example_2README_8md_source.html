94 MINIMAL-NET HOWTO

97 With the target minimal-net you can test your CoAP applications without
98 constraints, i.e., with large buffers, debug output, memory protection, etc.

126 DETAILS
127 -------
128 
129 Erbium implements the Proposed Standard of CoAP. Central features are commented
130 in er-example-server.c. In general, apps/er-coap supports:
131 
132 - All draft-18 header options
133 - CON Retransmissions (note COAP_MAX_OPEN_TRANSACTIONS)
134 - Blockwise Transfers (note REST_MAX_CHUNK_SIZE, see er-plugtest-server.c for
135  Block1 uploads)
136 - Separate Responses (no rest_set_pre_handler() required anymore, note
137  coap_separate_accept(), _reject(), and _resume())
138 - Resource Discovery
139 - Observing Resources (see EVENT_ and PRERIODIC_RESOURCE, note
140  COAP_MAX_OBSERVERS)
