1 Getting Started with Contiki using avr-rss2
2 ===========================================
3 This guide's aim is to help you start using Contiki for RSS2 boards
4 The platform supports different AtMega-RF boards:
5 
6 * Based on MCU AtMega128RFR2
7 * Based on MCU AtMega256RFR2
8 * Based on MCU AtMega128RFA1
9 
10 Boards all looks the same. AtMega256RFR2 boards are labeled with "Made in EU".
11 The platform and CPU code, are common for both boards, can be found under
12 `$(CONTIKI)/platfrom/avr-rssa`. The port was developed and tested with RSS2.
13 This guide assumes that you have basic understanding of how to use the
14 command line and perform basic admin tasks on UNIX family OSs. You should
15 also have understanding about the Contiki OS design as well have C
16 programming skills.
17 
18 Board Features
19 ----------------
20 * Chip Antenna. Supercardiod.
21 * Robust radio performance. Up to 300 meter line-of-sight.
22 * Unique EUI64 address via chip combined with EEPROM
23 * Onboard temp sensor DS18B20,
24 * Light Sensor.
25 * 32kHz RTC xtal
26 * Comparator chip and input.
27 * SW FET, (relay) for power on/off of sensors.
28 * DC input via LDO up to 25V.
29 * Standard. 6-pin TTL-USB header for FTDI cable for UART.
30 * PCB formfactor for cheap project box G.40X IP54
31 * Power/current:
32  * RX ~10mA (Full RPC AtMegaXXRFR2).
33  * Sleep ~45uA @ 16MHz XTAL
34  * Sleep ~15uA @ 8MHz using internal oscillator
35 * Preprogammed bootloader.
36 * CE certified by test institute.
37 
38 UART
39 ----
40 The board has one UART via the 6-pin TTL-USB adapter, The recommended
41 baudrate is 38400 bps. This speed gives the lowest error with respect
42 of the used clock frequency used internally. A possible hi-speed is
43 250000 bps wich gives 0% Error with 16MHz clock.

45 Port Features
46 --------------
47 The platform has the following key features:
48 * Standard, E64 address from built-in chip.
49 * First hooks for RPC (Reduced Power Consumption) for AtMegaXXXRFR2.

51 Toolchain needs
52 ---------------
53 The Atmel toolcahin is available in most operating system.
54 Note. The native OS packages does not yet support the new
55 AtMega256RfR2 MCU.
56 
57 We'll use toolchain from Atmels web-site. You might also consider

61 ###### For Linux
62 
63 1. See http://www.atmel.com/tools/ATMELAVRTOOLCHAINFORLINUX.aspx
64 2. Download the proper 8-bit platform 32 or 64 bit.
65 3. Unpack under `/usr/local`
66 4. Add to your search PATH. For example add to `.bashrc`: `export PATH=$PATH:/usr/local/avr8-gnu-toolchain-linux_x86_64/bin` (for 64 bit systems) or `export PATH=$PATH:/usr/local/avr8-gnu-toolchain-linux_x86/bin` (for 32 birt systems).
67 5. For flash programming you'll need `avrdude`. It can be installed with the command
68 `apt-get install avrdude`

86 Flashing
87 --------
88 Programming using avrdude using serial bootloader. (TTL-USB cable)
89 Press the RESET button. The bootloader with wait for boot commands
90 for 3 seconds.
91 
92 Flashing commnad line example 256k MCU:
93 
94  avrdude -p m256rfr2 -c stk500v2 -P /dev/ttyUSB0 -b 38400 -e -U flash:w:hello-world.avr-rss2
95 
96 Flashing commnad line example 128k MCU:
97 
98  avrdude -p m128rfa1 -c avr109 -P /dev/ttyUSB0 -b 38400 -e -U flash:w:hello-world.avr-rss2
99 
100 Older boards may use the avr109 boot loader. The `stk500v2` uses the yellow
101 lED while the avr109 uses the red LED.
