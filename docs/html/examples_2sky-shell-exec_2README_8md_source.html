1 ELFloader and shell command 'exec' example for Sky platform
2 -----------------------------------------------------------
3 
4 Compiles the Contiki hello-world application as a Contiki executable (.ce).
5 The Contiki executable is then uploaded to the Sky platform via serial, and is
6 stored in the filesystem. Finally, the executable is loaded via the shell
7 command 'exec'.
8 
9 NOTE: You may have to reduce the ELF loader memory usage
10 (/platform/sky/contiki-conf.h). Since hello-world uses very little memory:
11 
12 #define ELFLOADER_CONF_DATAMEMORY_SIZE 0x100
13 #define ELFLOADER_CONF_TEXTMEMORY_SIZE 0x100
14 
15 1. Upload Sky shell with 'exec' command and symbols (requires several
16  recompilations to generate correct symbols):
17 
18  make sky-shell-exec.sky
19  make sky-shell-exec.sky CORE=sky-shell-exec.sky
20  make sky-shell-exec.upload CORE=sky-shell-exec.sky
21 
22 2. Verify access to the shell and the filesystem:
23 
24  > make login
25  SHELL> echo hello shell
26  SHELL> echo test | write mytest.txt
27  SHELL> ls
28  SHELL> read mytest.txt
29  [CTRL-C] to exit the shell
30 
31 3. Upload Contiki executable hello-world.ce:
32 
33  > make upload-executable
34  [CTRL-C] to exit the shell when the entire file has been uploaded (after
35  ~30 sec)
36 
37 4. Verify that hello-world.ce exists in CFS:
38 
39  > make login
40  SHELL> ls
41  SHELL> read hello-world.ce | size
42 
43 The last command output should equal the size of hello-world.ce in this
44 directory!
45 
46 5. Load and start hello world:
47 
48  SHELL> exec hello-world.ce
49 
50 The program should now start: the output 'Hello, World' appears.
