1 MQTT Demo
2 =========
3 The MQTT client can be used to:
4 
5 * Publish sensor readings to an MQTT broker.
6 * Subscribe to a topic and receive commands from an MQTT broker
7 
8 The demo will give some visual feedback with the green LED:
9 * Very fast blinking: Searching for a network
10 * Fast blinking: Connecting to broker
11 * Slow, long blinking: Sending a publish message
12 
13 Publishing
14 ----------
15 By default the example will attempt to publish readings to an MQTT broker
16 running on the IPv6 address specified as `MQTT_DEMO_BROKER_IP_ADDR` in
17 `project-conf.h`. This functionality was tested successfully with
18 [mosquitto](http://mosquitto.org/).
19 
20 The publish messages include sensor readings but also some other information,
21 such as device uptime in seconds and a message sequence number. The demo will
22 publish to topic `iot-2/evt/status/fmt/json`. The device will connect using
23 client-id `d:quickstart:cc2538:<device-id>`, where `<device-id>` gets
24 constructed from the device's IEEE address.
25 
26 Subscribing
27 -----------
28 You can also subscribe to topics and receive commands, but this will only
29 work if you use "Org ID" != 'quickstart'. To achieve this, you will need to
30 change 'Org ID' (`DEFAULT_ORG_ID`). In this scenario, the device will subscribe
31 to:
32 
33 `iot-2/cmd/+/fmt/json`
34 
35 You can then use this to toggle LEDs. To do this, you can for example
36 use mosquitto client to publish to `iot-2/cmd/leds/fmt/json`. So, to change
37 the state of an LED, you would do this:
38 
39 `mosquitto_pub -h <broker IP> -m "1" -t iot-2/cmd/leds/fmt/json`
40 
41 Where `broker IP` should be replaced with the IP address of your mosquitto
42 broker (the one where you device has subscribed). Replace `-m "1'` with `-m "0"`
43 to turn the LED back off.
44 
45 Bear in mind that, even though the topic suggests that messages are of json
46 format, they are in fact not. This was done in order to avoid linking a json
47 parser into the firmware. This comment only applies to parsing incoming
48 messages, outgoing publish messages use proper json payload.
49 
50 IBM Quickstart Service
51 ----------------------
52 It is also possible to publish to IBM's quickstart service. To do so, you need
53 to undefine `MQTT_DEMO_BROKER_IP_ADDR`.
54 
55 The device will then try to connect to IBM's quickstart over NAT64, so you will
56 need a NAT64 gateway in your network to make this work. A guide on how to
57 setup NAT64 is out of scope here.
58 
59 If you want to use IBM's cloud service with a registered device, change
60 'Org ID' (`DEFAULT_ORG_ID`) and provide the 'Auth Token' (`DEFAULT_AUTH_TOKEN`),
61 which acts as a 'password', but bear in mind that it gets transported in clear
62 text.
