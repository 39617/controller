1 This README contains information how to establish an IPv6 connecton between
2 Linux BLE router and an IPSP enabled BLE device.

4 Prerequisites
5 =============
6 In general, any device capable of running Linux operating system, can be used
7 as a BLE router provided the following conditions are met:
8 
9 * Linux Kernel >3.18 is used
10 * bluez, libcap-ng0, radvd tools are present.
11 
12 If a built-in Bluetooth device is not available then Bluetooth 4.0 compatible
13 USB dongle can be used.
14 
15 The following procedures have been tested on Ubuntu 15.10.
16 
17 Establishing an IPv6 connection
18 ===============================
19 Use the following procedure to establish a connection between an nRF52 device
20 and Linux router:
21 
22 First enable 6LoWPAN module. This is neccessary only once per session:
23 
24  # Log in as a root user.
25  sudo su
26 
27  # Mount debugfs file system.
28  mount -t debugfs none /sys/kernel/debug
29 
30  # Load 6LoWPAN module.
31  modprobe bluetooth_6lowpan
32 
33  # Enable the bluetooth 6lowpan module.
34  echo 1 > /sys/kernel/debug/bluetooth/6lowpan_enable
35 
36  # Look for available HCI devices.
37  hciconfig
38 
39  # Reset HCI device - for example hci0 device.
40  hciconfig hci0 reset
41 
42  # Read 00:AA:BB:XX:YY:ZZ address of the nRF5x device.
43  hcitool lescan
44 
45 If you see device name and address in lescan output then you can connect to the
46 device:
47 
48  echo "connect 00:AA:BB:XX:YY:ZZ 1" > /sys/kernel/debug/bluetooth/6lowpan_control
49 
50 If above is successful then LED1 will stop blinking and LED2 will switch on.
51 You can then check the connection using the following commands:
52 
53  # Check if bt0 interface is present and up
54  ifconfig
55 
56  # Try to ping the device using its link-local address, for example, on bt0 interface.
57  ping6 -I bt0 fe80::2aa:bbff:fexx:yyzz
58 
59 If you'd like to learn more about the procedure please refer to
60 [Connecting devices to the router].
61 
62 Distributing routable IPv6 prefix
63 =================================
64 In Linux, Router Advertisement Daemon (RADVD) can be used to distribute prefixes
65 in the network, hance configure routable IPv6 address.
66 
67 To configure RADVD create `/etc/radvd.conf` file and paste the following contents:
68 
69  interface bt0
70  {
71  AdvSendAdvert on;
72  prefix 2001:db8::/64
73  {
74  AdvOnLink off;
75  AdvAutonomous on;
76  AdvRouterAddr on;
77  };
78  };
79 
80 Next, start RADVD daemon:
81 
82  # Set IPv6 forwarding (must be present).
83  sudo echo 1 > /proc/sys/net/ipv6/conf/all/forwarding
84  # Run radvd daemon.
85  sudo service radvd restart
86 
87 If successfull then all devices connected to the host will receive
88 a routable `2001:db8` prefix.
89 
90 This can be verified by sending echo request to the full address:
91 
92  ping6 -I bt0 2001:db8::2aa:bbff:fexx:yyzz
93 
94 where `aa:bbff:fexx:yyzz` is device Bluetooth address.
95 
96 If you'd like to learn more about the procedure please refer to
97 [Distributing a global IPv6 prefix].
