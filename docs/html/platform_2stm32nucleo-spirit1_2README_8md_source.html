18 Port Feature
19 ============
20 
21 The port supports the following boards from ST:
22 - NUCLEO-L152RE board, based on the STM32L152RET6 ultra-low power microcontroller
23 - X-NUCLEO-IDS01A4 based on sub-1GHz SPSGRF-868 SPIRIT1 module (operating at 868 MHz)
24 - X-NUCLEO-IDS01A5 based on sub-1GHz SPSGRF-915 SPIRIT1 module (operating at 915 MHz)
25 - X-NUCLEO-IKS01A1 featuring motion MEMS and environmental sensors (optional)
26 
27 The following drivers are included:
28 - LEDs and user button
29 - USB
30 - SPIRIT1 sub-1GHz transceiver
31 - HTS221, LIS3MDL, LPS25HB, LSM6DS0 sensors

41 Hardware Requirements
42 =====================
43 
44 * NUCLEO-L152RE development board
45 
46  >The NUCLEO-L152RE board belongs to the STM32 Nucleo family.
47 It features an STM32L152RET6 ultra-low power microcontroller based on ARM Cortex M3 MCU.
48 For detailed information on the NUCLEO-L152RE development board, please go to http://www.st.com and search for the "NUCLEO-L152RE" part number.
49 
50 
51 * X-NUCLEO-IDS01Ax sub-1GHz expansion board
52 
53  >The X-NUCLEO-IDS01A4 and X-NUCLEO-IDS01A5 are STM32 Nucleo expansion boards that use
54 the module SPSGRF-868 or SPSGRF-915 based on SPIRIT1 low data rate, low power sub-1 GHz transceiver.
55 
56  >The user can select the X-NUCLEO-IDS01A4 board to operate the SPIRIT1 transceiver at 868MHz or the X-NUCLEO-IDS01A5 board to operate the SPIRIT1 transceiver at 915MHz.
57 
58 >For detailed information on the X-NUCLEO-IDS01A4 or X-NUCLEO-IDS01A5 expansion board, or the SPIRIT1 transceiver please go to http://www.st.com and search for the specific part number.
59 
60 
61 * X-NUCLEO-IKS01A1, motion MEMS and environmental sensors expansion board (OPTIONAL)
62 
63  >The X-NUCLEO-IKS01A1 is a motion MEMS and environmental sensor evaluation board.
64 The use of this board is optional in the stm32nucleo-spirit1 Contiki platform.
65 
66  >For detailed information on the X-NUCLEO-IKS01A1 expansion board, please go to http://www.st.com and search for the "X-NUCLEO-IKS01A1" part number.

71 Software Requirements
72 =====================
73 
74 The following software are needed:
75 
76 * ST port of Contiki for STM32 Nucleo and expansion boards.
77 
78  >The port is automatically installed when both the Contiki and the submodule repository are cloned: the former hosts the Contiki distribution and the ST platform interface, the latter hosts the actual library. The following command downloads the full porting:
79 
80  git clone --recursive https://github.com/contiki-os/contiki
81 
82 Alternatively, if you cloned the contiki repository without the "--recursive" option, you can simply download the submodule repository with the following commands:
83 
84  cd contiki/
85  git submodule init
86  git submodule update
87 
88 The platform name is: stm32nucleo-spirit1
89 
90 * A toolchain to build the firmware: The port has been developed and tested with GNU Tools
91 for ARM Embedded Processors.
92  >The toolchain can be found at: https://launchpad.net/gcc-arm-embedded
93 The port was developed and tested using this version: gcc-arm-none-eabi v4.8.3

105 Build an example
106 ================
107 In order to build an example go to the selected example directory (see a list of tested
108 examples in the previous section).
109 
110 For example, go to examples/ipv6/simple-udp-rpl directory.
111 
112 
113 If the X-NUCLEO-IDS01A4 sub-1GHz RF expansion board is used, the following must be run:
114 
115  make TARGET=stm32nucleo-spirit1 BOARD=ids01a4 clean
116  make TARGET=stm32nucleo-spirit1 BOARD=ids01a4
117 
118 If the X-NUCLEO-IDS01A5 sub-1GHz RF expansion board is used, the following must be run:
119 
120  make TARGET=stm32nucleo-spirit1 BOARD=ids01a5 clean
121  make TARGET=stm32nucleo-spirit1 BOARD=ids01a5
122 
123 
124 This will create executables for UDP sender and receiver nodes.
125 
126 In order to generate binary files that can be flashed on the STM32 Nucleo the following command must be run:
127 
128  arm-none-eabi-objcopy -O binary unicast-sender.stm32nucleo-spirit1 unicast-sender.bin
129  arm-none-eabi-objcopy -O binary unicast-receiver.stm32nucleo-spirit1 unicast-receiver.bin
130 
131 These executables can be programmed on the nodes using the procedure described hereafter.
132 
133 
134 In case you need to build an example that uses the additional sensors expansion board
135 (for example, considering a system made of NUCLEO-L152RE, X-NUCLEO-IDS01A4 and X-NUCLEO-IKS01A1)
136 then the command to be run would be:
137 
138  make TARGET=stm32nucleo-spirit1 BOARD=ids01a4 SENSORBOARD=iks01a1

140 System setup
141 ============
142 
143 1. Check that the jumper on the J1 connector on the X-NUCLEO-IDS01Ax expansion board is connected.
144 This jumper provides the required voltage to the devices on the board.
145 
146 2. Connect the X-NUCLEO-IDS01Ax board to the STM32 Nucleo board (NUCLEO-L152RE) from the top.
147 
148 3. If the optional X-NUCLEO-IKS01A1 board is used, connect it on top of the X-NUCLEO-IDS01Ax board.
149 
150 4. Power the STM32 Nucleo board using the Mini-B USB cable connected to the PC.
151 
152 5. Program the firmware on the STM32 Nucleo board.
153 This can be done by copying the binary file on the USB mass storage that is
154 automatically created when plugging the STM32 Nucleo board to the PC.
155 
156 6. Reset the MCU by using the reset button on the STM32 Nucleo board
