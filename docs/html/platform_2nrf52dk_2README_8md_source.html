1 Contiki for nRF52 Development Kit
2 =================================
3 This guide's aim is to help you with using Contiki for
4 Nordic Semiconductor's nRF52 DK.
5 
6 The port depends on Nordic Semiconductor IoT SDK for nRF52.
7 The IoT SDK contains source code and libraries which are
8 required for successfull port compilation. It also contains
9 SoftDevice binary driver which is required for BLE operation.
10 See prerequisites section for details on how to set up the SDK.
11 
12 For more information about SoftDevice please refer to the SDK
13 docummentation [nRF52 Datasheet and SDK documentation].
14 
15 This port supports DK versions PCA10040 and PCA10036.
16 
17 Port Features
18 =============
19 The following features have been implemented:
20 * Support for IPv6 over BLE using Contiki 6LoWPAN implementation
21 * Contiki system clock and rtimers (using 32kHz and 1MHz timers)
22 * UART driver
23 * Watchdog driver
24 * Hardware RNG
25 * Temperature sensor driver
26 * DK LED driver
27 * DK Buttons driver
28 * Real Time Transfer (RTT) I/O support
29 
30 Note that this port supports only IPv6 network stack.
31 
32 The port is organized as follows:
33 * nRF52832 CPU and BLE drivers are located in `cpu/nrf52832` folder
34 * nRF52 Development Kit drivers are located in `platform/nrf52dk` folder
35 * Platform examples are located in `examples/nrf52dk` folder
36 
37 Prerequisites and Setup
38 =======================
39 In order to compile for the nRF52 DK platform you'll need:
40 
41 * nRF5 IOT SDK
42  https://developer.nordicsemi.com
43 
44  Download nRF5 IOT SDK, extract it to a folder of your choice,
45  and point `NRF52_SDK_ROOT` environmental variable to it, e.g.,:
46 
47  ```
48  wget https://developer.nordicsemi.com/nRF5_IoT_SDK/nRF5_IoT_SDK_v0.9.x/nrf5_iot_sdk_3288530.zip
49  unzip nrf5_iot_sdk_3288530.zip -d /path/to/sdk
50  export NRF52_SDK_ROOT=/path/to/sdk
51  ```
52 
53 * An ARM compatible toolchain
54  The port has been tested with GNU Tools for ARM Embedded Processors
55  version 5.2.1.
56 
57  For Ubuntu you can use package version provided by your distribution:
58  ```
59  sudo apt-get install gcc-arm-none-eabi
60  ```
61 
62  Alternatively, install the toolchain from PPA to get the latest version
63  of the compiler: https://launchpad.net/~team-gcc-arm-embedded/+archive/ubuntu/ppa
64 
65  For other systems please download and install toolchain available at
66  https://launchpad.net/gcc-arm-embedded
67 
68 * GNU make
69 
70 * Segger JLink Software for Linux
71  https://www.segger.com/jlink-software.html
72 
73  This package contains tools necessary for programming and debugging nRF52 DK.
74 
75  For Ubuntu you can download and install a .deb package. Alternatively download
76  tar.gz archive and extract it to a folder of your choice. In this case you
77  need to set `NRF52_JLINK_PATH` environmental variable to point to the
78  JLink tools location:
79 
80  ```
81  export NRF52_JLINK_PATH=/path/to/jlink/tools
82  ```
83 
84  To keep this variable set between sessions please add the above line to your
85  `rc.local` file.
86 
87  In order to access the DK as a regular Linux user create a `99-jlink.rules`
88  file in your udev rules folder (e.g., `/etc/udev/rules.d/`) and add the
89  following line to it:
90 
91  ```
92  ATTRS{idProduct}=="1015", ATTRS{idVendor}=="1366", MODE="0666"
93  ```
94  When installing from a deb package, the `99-jlink.rules` file is added
95  automatically to /etc/udev/rules.d folder. However, the syntax of the file
96  doesn't work on newer udev versions. To fix this problem edit this file and
97  replace ATTR keyword with ATTRS.
98 
99 To fully use the platform a BLE enabled router device is needed. Please refer
100 to `Preqrequisites` section in `README-BLE-6LoWPAN.md` for details.

102 Getting Started
103 ===============
104 Once all tools are installed it is recommended to start by compiling
105 and flashing `examples/hello-word` application. This allows to verify
106 that toolchain setup is correct.
107 
108 To compile the example, go to `examples/hello-world` and execute:
109 
110  make TARGET=nrf52dk
111 
112 If you haven't used the device with Contiki before we advise to
113 erase the device and flash new SoftDevice:
114 
115  make TARGET=nrf52dk erase
116  make TARGET=nrf52dk softdevice.flash
117 
118 If the compilation is completed without errors flash the board:
119 
120  make TARGET=nrf52dk hello-world.flash
121 
122 The device will start BLE advertising as soon as initialized. By
123 default the device name is set to 'Contiki nRF52 DK'. To verify
124 that the device is advertising properly run:
125 
126  sudo hcitool lescan
127 
128 And observe if the device name appears in the output. Also, observe
129 if LED1 is blinking what indicates that device is waiting for a connection
130 from BLE master.
131 
132 If device is functioning as expected you can test IPv6 connection
133 to the device. Please refer to `README-BLE-6LoWPAN.md` on details how to do
134 this.

144 Compilation Options
145 ===================
146 The Contiki TARGET name for this port is `nrf52dk`, so in order to compile
147 an application you need to invoke GNU make as follows:
148 
149  make TARGET=nrf52dk
150 
151 In addition to this port supports the following variables which can be
152 set on the compilation command line:
153 
154 * `NRF52_SDK_ROOT=<SDK PATH>`
155  This variable allows to specify a path to the nRF52 SDK which should
156  be used for the build.
157 
158 * `NRF52_WITHOUT_SOFTDEVICE={0|1}`
159  Disables SoftDevice support if set to 1. By default, SoftDevice support
160  is used. Note that SoftDevice must be present (flashed) in the device
161  before you run an application that requires it's presence.
162 
163 * `NRF52_USE_RTT={0|1}`
164  Enables RealTime Terminal I/O. See VCOM and RTT for details. By default,
165  RTT is disabled and IO is done using Virtual COM port.
166 
167 * `NRF52_JLINK_SN=<serial number>`
168  Allows to choose a particular DK by its serial number (printed on the
169  label). This is useful if you have more than one DK connected to your
170  PC and whish to flash a particular device.
171 
172 * `NRF52_DK_REVISION={pca10040|pca10036}`
173  Allows to specify DK revision. By default, pca10040 is used.
174 
175 Compilation Targets
176 ===================
177 Invoking make solely with the `TARGET` variable set will build all
178 applications in a given folder. A particular application can be built
179 by invoking make with its name as a compilation target:
180 
181  make TARGET=nrf52dk hello-world
182 
183 In order to flash the application binary to the device use `<application>.flash`
184 as make target, e.g.:
185 
186  make TARGET=nrf52dk hello-world.flash
187 
188 In addition, the SoftDevice binary can be flashed to the DK by invoking:
189 
190  make TARGET=nrf52dk softdevice.flash
191 
192 To remove all build results invoke:
193 
194  make TARGET=nrf52dk clean
195 
196 The device memory can be erased using:
197 
198  make TARGET=nrf52dk erase
199 
200 Note, that once the device is erased, the SoftDevice must be programmed again.

202 Virtual COM and Real Time Transfer
203 ==================================
204 By default, the nRF52 DK uses a Virtual COM port to output logs. Once
205 the DK is plugged in a `/tty/ACM<n>` or `/ttyUSB<n>` device should appear in
206 your filesystem. A terminal emulator, such as picocom or minicom, can be
207 used to connect to the device. Default serial port speed is 38400 bps.
208 
209 To connect to serial port using picocom invoke:
210 
211  picocom -fh -b 38400 --imap lfcrlf /dev/ttyACM0
212 
213 Note, that if you have not fixed file permissions for `/dev/ttyACM0`
214 according to section `Segger JLink Software for Linux` you'll need to use
215 root or sudo to open the port with `picocom`.
216 
217 In addition to Virtual COM the port supports SEGGER's Real Time Transfer
218 for low overhead I/O support. This allows for outputting debugging information
219 at much higher rate with significantly lower overhead than regular I/O.
220 
221 To compile an application with RTT rather that VCOM set `NRF52_USE_RTT` to 1 on
222 the compilation command line:
223 
224  make TARGET=nrf52dk NRF52_USE_RTT=1 hello-world
225 
226 You can then connect to the device terminal using `JLinkRTTClient`. Note that
227 a JLlink gdb or commander must be connected to the target for the RTT to work.
228 
229 More details regarding RTT can be found at https://www.segger.com/jlink-rtt.html
