1 Galileo Specific Examples
2 =========================
3 
4 This directory contains galileo-specific example applications to illustrate
5 how to use galileo APIs.
6 
7 In order to build a application, you should set the EXAMPLE environment
8 variable to the name of the application you want to build. For instance, if
9 you want to build gpio-output application, run the following command:
10 
11 ```
12 $ make TARGET=galileo EXAMPLE=gpio-output
13 ```
14 
15 The corresponding EXAMPLE variable setting for each application is
16 listed to the right of its heading.
17 
18 GPIO
19 ----
20 
21 ### GPIO Output (EXAMPLE=gpio-output)
22 
23 This application shows how to use the GPIO driver APIs to manipulate output
24 pins. This application sets the GPIO 5 pin as output pin and toggles its
25 state at every half second.
26 
27 For a visual effect, you should wire shield pin IO2 to a led in a protoboard.
28 Once the application is running, you should see a blinking LED.
29 
30 ### GPIO Input (EXAMPLE=gpio-input)
31 
32 This application shows how to use the GPIO driver APIs to manipulate
33 input pins. This application uses default galileo pinmux
34 initialization and sets the GPIO 5 (shield pin IO2) as output pin and
35 GPIO 6 (shield pin IO3) as input. A jumper should be used to connect
36 the two pins. The application toggles the output pin state at every
37 half second and checks the value on input pin.
38 
39 ### GPIO Interrupt (EXAMPLE=gpio-interrupt)
40 
41 This application shows how to use the GPIO driver APIs to manipulate
42 interrupt pins. This application uses default galileo pinmux
43 initialization and sets the GPIO 5 (shield pin IO2) as output pin and
44 GPIO 6 (shield pin IO3) as interrupt. A jumper should be used to
45 connect the two pins. It toggles the output pin stat at every half
46 second in order to emulate an interrupt. This triggers an interrupt
47 and the application callback is called. You can confirm that though
48 the UART output.

53 ### I2C LSM9DS0 (EXAMPLE=i2c-LSM9DS0)
54 
55 This application shows how to use I2C driver APIs to configure I2C
56 Master controller and communicate with an LSM9DS0 sensor if one has
57 been connected as described below. At every 5 seconds, the application
58 reads the "who am I" register from gyroscope sensor and prints if the
59 register value matches the expected value described in the spec [1].
60 
61 According to the sensor spec, to read the value in "who am I" register, we
62 should first perform an i2c write operation to select the register we want
63 to read from and then we perform the i2c read operation to actually read
64 the register contents.
65 
66 The wiring setup is as follows (left column from Galileo and right column from LSM9DS0):
67 - 3.3v and Vin
68 - GND and GND
69 - GND and SDOG
70 - 3.3v and CSG
71 - SDA and SDA
72 - SCL and SCL
73 
74 ### I2C Callbacks (EXAMPLE=i2c-callbacks)
75 
76 This application is very similar to the previous one in that it also
77 shows how to use I2C callback functionality, but it can be run without
78 attaching any additional sensors to the platform since it simply
79 communicates with a built-in PWM controller.
80 
81 Every five seconds, the application reads the current value of the
82 MODE1 register, which should have previously been initialized to the
83 value 0x20. The test verifies that this expected value is returned by
84 the read.

89 ### Print IMR info (EXAMPLE=print-imr)
90 
91 This application prints out information about the configuration of the
92 Intel Quark X1000 SoC Isolated Memory Regions (IMRs), the Host System
93 Management Mode Controls register, and the Host Memory I/O Boundary

99 ### Protection Domain Switch Latency (EXAMPLE=prot-domain-switch-latency)
100 
101 This application measures and prints the average latency of repeatedly
102 switching from one protection domain to another and back, in ping-pong
103 fashion. It can optionally perform memory accesses to metadata
104 associated with the destination protection domain. This feature can
105 be enabled by specifying SAMPLE_METADATA=1 on the build command line.
