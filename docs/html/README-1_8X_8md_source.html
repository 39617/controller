1 Win32
2 =====
3 
4 The platform/win32/ directory contains a showcase of Contiki 1.x technologies
5 ported to the Contiki 2.x environment. As such it serves primarily two
6 purposes:
7 
8 - Allow for easy interactive experience of the Contiki applications with user
9  interface, especially the Contiki web browser.
10 - Allow for easy regression tests of the code in core/ctk/ and most of the code
11  in apps/.
12 
13 The employed Contiki 1.x technologies include:
14 
15 - The 'Contiki Tool Kit' (CTK) UI framework in general
16 - CTK running in a character based environment (here the Win32 Console API)
17 - CTK mouse support
18 - Dynamic loading and unloading of Contiki programs (.PRG) and Contiki program
19  descriptions (.DSC) based on dynamically loadable libraries (here Win32 DLLs)
20 
21 A typical Contiki 1.x implementation consists of a core binary (containing
22 among others the uIP and CTK libraries) and the program (and program
23 description) binaries which are both loaded by the core and reference the core
24 libraries. Therefore the Contiki 2.x project-based build system doesn't suit
25 exactly. So in order to be able to leverage the Contiki 2.x build system to its
26 maximum extend two makefiles are necessary:
27 
28 - Makefile plays the role of a Contiki 2.x project Makefile
29 - Makefile.win32 is an ordinary (yet complex) Contiki 2.x Makefile.$(TARGET)
30 
31 As platform/win32/Makefile.win32 includes cpu/native/Makefile.native the
32 WinPcap library is used for network I/O. Please consult
33 cpu/native/net/README-WPCAP.md for further details.
34 
35 Both a Cygwin Bash Shell and an ordinary Windows Command Prompt are fine for
36 runtime. But in order to experience the CTK mouse support it is necessary to
37 deactivate the 'QuickEdit Mode' of the window used. This option is found in its
38 Windows system menu under 'Properties'.
39 
40 As the console is used for displaying the actual user interface the log output
41 is routed to the debug output. DebugView is a very lean program for displaying
42 the debug output in case no debugger is active and does so. It is available at
43 [http://technet.microsoft.com/en-us/sysinternals/bb896647.aspx](http://technet.microsoft.com/en-us/sysinternals/bb896647.aspx).
