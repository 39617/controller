1 6502
2 ====
3 
4 The cpu/6502/ directory is used for targeting 6502-based machines using the
5 cc65 compiler [http://cc65.github.io/cc65/](http://cc65.github.io/cc65/).
6 
7 The Contiki network configuration for 6502-based targets is loaded from a
8 binary configuration file (by default named contiki.cfg). It has the following
9 format for Ethernet:
10 
11 - Bytes 1 - 4: IP Address (HiByte first)
12 - Bytes 5 - 8: Subnet Mask (HiByte first)
13 - Bytes 9 - 12: Default Router (HiByte first)
14 - Bytes 13 - 16: DNS Server (HiByte first)
15 - Bytes 17 - 18: Ethernet card I/O address (LoByte first !)
16 - Bytes 19 - xx: Ethernet card driver name (ASCII / PETSCII)
17 
18 It has the following format for SLIP (based on RS232 driver coming with cc65):
19 
20 - Bytes 1 - 4: IP Address (HiByte first)
21 - Bytes 5 - 8: Subnet Mask (HiByte first)
22 - Bytes 9 - 12: Default Router (HiByte first)
23 - Bytes 13 - 16: DNS Server (HiByte first)
24 - Bytes 17 - 21: struct ser_params (see cc65 serial.h)
25 
26 The build for 6502-based machines includes the 'disk' make goal which creates a
27 bootable floppy disk image containing the project binary, a sample
28 configuration file and the Ethernet card drivers.
29 
30 The build for 6502-based machines supports so-called high-level configuration
31 macros which allow to customize Contiki on a per-project basis. They are set in
32 form of a comma-separated list as value of the make variable DEFINES on the
33 make command line. The value of DEFINES can be saved with the 'savedefines'
34 make goal. The values of the high-level configuration macros are not tracked by
35 the build so a manual rebuild is necessary on any change. The following
36 high-level configuration macros may be set:
37 
38 - WITH_SLIP
39  - Default: 0
40  - Purpose: Use SLIP (based on RS232 driver coming with cc65) instead of
41  Ethernet.
42 
43 - MTU_SIZE
44  - Default: 1500
45  - Purpose: Set the Maximum Transfer Unit size.
46 
47 - CONNECTIONS
48  - Default: 2
49  - Purpose: Set the maximum number of concurrent TCP connections.
50 
51 - STATIC_DRIVER
52  - Default: N/A
53  - Purpose: Link device driver statically instead of loading it dynamically
54  using the network configuration file.
55 
56 - WITH_LOGGING
57  - Default: 0
58  - Purpose: Have log_message() and UIP_LOG() write messages to the screen.
59 
60 - WITH_BOOST
61  - Default: 0
62  - Purpose: Significantly improve troughput on sending full sized packets by
63  splitting them thus workarounding the "delayed acknowledge".
64 
65 - WITH_FORWARDING
66  - Default: 0
67  - Purpose: Enable support for the 'IP forwarding' packet driver.
68 
69 - WITH_CLIENT
70  - Default: 0
71  - Purpose: Enable support for outgoing TCP connections.
72 
73 - WITH_DNS
74  - Default: 0
75  - Purpose: Enable UDP support and initialize resolver process on startup.
76 
77 - WITH_80COL
78  - Default: 0
79  - Purpose: Enable 80 column screen.
80 
81 - WITH_GUI
82  - Default: 0
83  - Purpose: Initialize the CTK process on startup.
84 
85 - WITH_MOUSE
86  - Default: 0
87  - Purpose: Enable CTK mouse support and load a mouse driver.
88 
89 - STATIC_MOUSE
90  - Default: N/A
91  - Purpose: Link mouse driver statically instead of loading it dynamically.
92 
93 - WITH_ARGS
94  - Default: 0
95  - Purpose: Enable support for contiki_argc / contiki_argv.
96 
97 - WITH_PFS
98  - Default: 0
99  - Purpose: Implement the CFS interface with a Platform-specific File System
100  instead of the POSIX file system.
