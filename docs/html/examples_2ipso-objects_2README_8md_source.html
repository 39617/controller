1 IPSO Objects Example
2 ============================================
3 
4 This is an example of how to make use of the IPSO Object and LWM2M
5 implementation in Contiki.
6 
7 The LWM2M implementation is based on the Erbium CoAP implementation
8 and consists of two apps: lwm2m-engine and ipso-objects. The
9 lwm2m-engine handle the specifics of LWM2M including bootstrapping and
10 how read/writes of objects and resources are handled. The ipso-objects
11 contains implementations of some of the IPSO Smart Objects.
12 
13 The implementation was used during the IPSO Interop in May 2015,
14 Kista, Sweden, and was successfully tested with other
15 implementations.
16 
17 The examples use some of the basic IPSO object for controlling LEDs on
18 Contiki devices and for reading out temperature.
19 
20 ##Testing IPSO-objects with Leshan
21 
22 First program a device with the examples/ipso-objects/example-ipso-objects.c
23 
24 ```bash
25 >make example-ipso-objects.upload TARGET=zoul
26 >...
27 ```
28 
29 After that start up a native-border router or other border router on fd00::1/64
30 or another prefix - NOTE: if you use another prefix you will need to change LWM2M_SERVER_ADDRESS for which the device will register - in project-conf.h:
31 ```
32 #define LWM2M_SERVER_ADDRESS "fd00::1"
33 ```
34 
35 Then when everything is setup you can download a Leshan and use that to
36 test controlling LEDs of the device.
37 
38 ###Starting Leshan
39 ```bash
40 wget https://hudson.eclipse.org/leshan/job/leshan/lastSuccessfulBuild/artifact/leshan-standalone.jar
41 java -jar ./leshan-standalone.jar
42 ```
43 Browse to leshans device page with http://127.0.0.1:8080 .
44 
45 When you have started the border-router and also Leshan you should now
46 start (or reboot) your IPSO Object enabled device. Within 30 seconds
47 you should be able to see it on the Leshan device page.
