1 hexameter : Convert Intel hex files to a binary file
2 ====================================================
3 
4 version 2.1.0 Copyright (c) 2003-2008, Takahide Matsutsuka.
5 
6 What is it?
7 -----------
8 
9 It converts Intel hex files, which emitted by SDCC-Z80 compiler, to a binary
10 file. You can attach additional prefix and/or suffix file to comply the file
11 with arbitrary binary format. It provides a development environment of C
12 language on PC-6001 and other old computers.
13 
14 Installation
15 ------------
16 
17 Installation can be done in the following steps.
18 
19 1. Download and install SDCC from [http://sdcc.sf.net](http://sdcc.sf.net)
20  Version 2.8.0 has been tested. SDCC is a cross-compiler for Z80 and other
21  8bit CPUs. Extract an archive on your disk, say "c:/sdcc".
22 
23 2. Place hexameter.exe and prefix/suffix files to any path-available directory.
24 
25 Use
26 ---
27 
28 1. Write your own C code.
29 
30 2. Compile your code using SDCC. While linking, you need to specify options
31  for SDCC so that the code can be worked on your machine. See
32  sample/Makefile for actual usage.
33 
34  sdcc -mz80 -c YOUR_CODE1.c
35  sdcc -mz80 -c YOUR_CODE2.c
36 
37  This step creates a file YOUR_CODE1.o and YOUR_CODE2.o respectively, which
38  run on Z80-based machine.
39 
40  for target in YOUR_CODE1.o YOUR_CODE2.o; do echo $@ >> YOUR_LIB.lib $@; done
41 
42  This step makes a library file.
43 
44  sdcc -mz80 --no-std-crt0 --code-loc 0x840f --data-loc 0 -o YOUR_APP.ihx crt0.o -lYOUR_LIB.lib
45 
46  - -mz80 specifies the Z80 mode.
47  - --code-loc specifies the start address of your object code. Basically, it
48  depends on the machine and RAM size. As for PC-6001, 0x840f for 32kB, or
49  0xc40f for 16kB. As for PC-6001mkII, it would be 0x800f. If you are not
50  sure what you are about to do, just leave it as default 0x840f.
51  - --data-loc 0 specifies the code is followed by data. You can specify an
52  arbitrary address instead.
53  - --no-std-crt0 indicates that you use your own crt0 instead of sdcc's
54  default crt0 object. The customized crt0.o file is in lib directory of
55  this release.
56 
57 3. Convert ihx file to cas file using hexameter
58 
59  hexameter <ihx_file|binary_file> [ihx_file|binary_file ...]
60 
61  The ihx files are just attached in the specified order.
62 
63  Here you can take some options:
64  -o <filename> specify output file name
65  -d <name>=<value> define ihx-specific value replacement
66  -v verbose output
67  -b <size> size of the output file in hexadecimal bytes.
68  only if the size of the output is less than the size
69  specified, the trailing bytes will filled by zeroes.
70  note that it doesn't mean the size of output is
71  restricted by the given size.
72  -h displays simple usage
73 
74  You can find various predefined library file in "lib" directory.
75  Each of them may take its own optional value, which you can specify
76  with -d option.
77 
78 4. Example
79 
80  The following is a typical example to convert from ihx files to
81  PC-6001 loadable cassette format.
82 
83  hexameter -v TAPE=myfile mode2.ihx mycode2.ihx -o myapp.p6
84 
85  Note that TAPE value is defined in mode2.ihx, defines cassette file name.
86 
87 5. Load your cas file into your 6001 emulator.
88 
89  I've checked it working on the following emulators:
90 
91  [iP6Win](http://www.retropc.net/mm/pc6001)
92 
93 IHX extentions
94 --------------
95 
96 To support run-time user specified values in ihx files, Hexameter supports
97 extended ihx files. Examples are located in ihx directory.
98 
99 For example, mode2.ihx has the following line.
100 
101  :06000a02TAPE
102 
103 As in normal ihx format, the first 9 characters conform to the following
104 format:
105 
106  :AABBBBCC
107  AA bytes encoded in this line
108  BBBB start address of this line
109  CC type of this line
110 
111 The normal ihx file, which sdcc emits, has the type 00 (data) and 01 (end of
112 file). In addition to this, Hexameter supports the following types.
113 
114  02 string
115  03 byte
116  04 word (encoded in little endian)
117 
118 The rest of line defines a name of the definition. In the example above, the
119 name TAPE is assigned to this line. You can use latin alphabets, numbers, and
120 underscore for the name. Letters are case-sensitive.
121 
122 When you run Hexameter, you can specify a value for this definition like:
123 
124  hexameter -d TAPE=xxxx
125 
126 Since this example defines 06 in the bytes field, TAPE can have up to six
127 characters. This string will override the memory location 000a specified in
128 the second field.
129 
130 For type 03 (byte) and 04 (word), bytes field has no effect.
