1 The Contiki build system
2 ========================
3 
4 The Contiki build system is designed to make it easy to compile Contiki
5 applications for different hardware platforms or into a simulation platform by
6 simply supplying different parameters to the make command, without having to
7 edit makefiles or modify the application code.
8 
9 The file example project in examples/hello-world/ shows how the Contiki build
10 system works. The hello-world.c application can be built into a complete
11 Contiki system by running make in the examples/hello-world/ directory. Running
12 make without parameters will build a Contiki system using the native target.
13 The native target is a special Contiki platform that builds an entire Contiki
14 system as a program that runs on the development system. After compiling the
15 application for the native target it is possible to run the Contiki system with
16 the application by running the file hello-world.native. To compile the
17 application and a Contiki system for the ESB platform the command make
18 TARGET=esb is used. This produces a hello-world.esb file that can be loaded
19 into an ESB board.
20 
21 To compile the hello-world application into a stand-alone executable that can
22 be loaded into a running Contiki system, the command make hello-world.ce is
23 used. To build an executable file for the ESB platform, make TARGET=esb
24 hello-world.ce is run.
25 
26 To avoid having to type TARGET= every time make is run, it is possible to run
27 make TARGET=esb savetarget to save the selected target as the default target
28 platform for subsequent invocations of make. A file called Makefile.target
29 containing the currently saved target is saved in the project's directory.
30 
31 Beside TARGET= there's DEFINES= which allows to set arbitrary variables for the
32 C preprocessor in form of a comma-separated list. Again it is possible to avoid
33 having to re-type i.e. DEFINES=MYTRACE,MYVALUE=4711 by running make TARGET=esb
34 DEFINES=MYTRACE,MYVALUE=4711 savedefines. A file called Makefile.esb.defines is
35 saved in the project's directory containing the currently saved defines for the
36 ESB platform.
37 
38 Makefiles used in the Contiki build system The Contiki build system is composed
39 of a number of Makefiles. These are:
40 
41 * Makefile: the project's makefile, located in the project directory.
42 
43 * Makefile.include: the system-wide Contiki makefile, located in the root of
44  the Contiki source tree.
45 
46 * Makefile.$(TARGET) (where $(TARGET) is the name of the platform that is
47  currently being built): rules for the specific platform, located in the
48  platform's subdirectory in the platform/ directory.
49 
50 * Makefile.$(CPU) (where $(CPU) is the name of the CPU or microcontroller
51  architecture used on the platform for which Contiki is built): rules for the
52  CPU architecture, located in the CPU architecture's subdirectory in the cpu/
53  directory.
54 
55 * Makefile.$(APP) (where $(APP) is the name of an application in the apps/
56  directory): rules for applications in the apps/ directories. Each application
57  has its own makefile.
58 
59 The Makefile in the project's directory is intentionally simple. It specifies
60 where the Contiki source code resides in the system and includes the
61 system-wide Makefile, Makefile.include. The project's makefile can also define
62 in the APPS variable a list of applications from the apps/ directory that
63 should be included in the Contiki system. The Makefile used in the hello-world
64 example project looks like this:
65 
66  CONTIKI_PROJECT = hello-world
67  all: $(CONTIKI_PROJECT)
68 
69  CONTIKI = ../..
70  include $(CONTIKI)/Makefile.include
71 
72 First, the location of the Contiki source code tree is given by defining the
73 CONTIKI variable. Next, the name of the application is defined. Finally, the
74 system-wide Makefile.include is included.
75 
76 The Makefile.include contains definitions of the C files of the core Contiki
77 system. Makefile.include always reside in the root of the Contiki source tree.
78 When make is run, Makefile.include includes the Makefile.$(TARGET) as well as
79 all makefiles for the applications in the APPS list (which is specified by the
80 project's Makefile).
81 
82 Makefile.$(TARGET), which is located in the platform/$(TARGET)/ directory,
83 contains the list of C files that the platform adds to the Contiki system. This
84 list is defined by the CONTIKI_TARGET_SOURCEFILES variable. The
85 Makefile.$(TARGET) also includes the Makefile.$(CPU) from the cpu/$(CPU)/
86 directory.
87 
88 The Makefile.$(CPU) typically contains definitions for the C compiler used for
89 the particular CPU. If multiple C compilers are used, the Makefile.$(CPU) can
90 either contain a conditional expression that allows different C compilers to be
91 defined, or it can be completely overridden by the platform specific makefile
92 Makefile.$(TARGET).
