1 Example: mockup of DC converter functionality for IoT-grid.
2 ===========================================
3 This example imitates DC converter functionality for IoT-grid.
4 We use standard IoT protocol stack with CoAP application-level protocol.
5 The settings are as follows.
6 
7 * APPLICATION: CoAP
8 * TRANSPORT: UDP
9 * NETWORK: IPv6/RPL
10 * ADAPTATION: 6LoWPAN
11 * MAC: nullmac_driver
12 * RADIO DUTY CYCLE: nullrdc_driver
13 * PHYSICAL: IEEE 802.15.4
14 
15 CoAP resources
16 ----------------
17 We define a CoAP resource for each functionality of DC converter.
18 Since each functionality may have several parameters, we define each resource
19 as a vector of parameters as follows.
20 
21 * /dcdc/status read-only parameters for power monitoring.
22 It also support periodic monitoring through CoAP observe option.
23  ** 0 VOUT Output voltage
24  ** 1 VIN Input voltage
25  ** 2 IOUT Output current
26  ** 3 IIN Input current
27 
28 * /dcdc/vdc configurable parameters for voltage droop control function.
29  ** 0 VGRID Desired grid output voltage
30  ** 1 SLOPE Slope of voltage droop control function
31  ** 2 PMAX Maximum output power allowed
32 
33 * /dcdc/hwcfg configurable parameters for DC converter hardware.
34  ** 0 VMAX Maximum output voltage allowed
35  ** 1 IMAX Maximum output current allowed
36 
37 Each functionality is implemented as a sensor type device.
38 They are located in "dev" folder.
39 * dc-status-sensor for /dcdc/status
40 * dc-vdc-sensor for /dcdc/vdc
41 * dc-hw-sensor for /dcdc/hwcfg
42 
43 The corresponding CoAP handler for each resource is defined in "resources" folder.
44 * res-dc-status-obs for /dcdc/status
45 * res-dc-vdc for /dcdc/vdc
46 * res-dc-hwcfg for /dcdc/hwcfg
47 
48 coap-server.c
49 ----------------
50 The server acts as the RPL root node.
51 It has 3 CoAP resources as described above.
52 
53 coap-client.c
54 ----------------
55 The client periodically send a CoAP command (either GET or PUT) to monitor/update
56 values of parameters of a resource.
57 
58 project-conf.h
59 ----------------
60 This file contains definitions needed for this IoT-grid example as follows.
61 * Enable IPv6 network stack
62 * Set nullrdc_driver
63 * Set nullmac_driver
64 * Activate CoAP observe client library (COAP_OBSERVE_CLIENT = 1)
65 * Increase the maximum number of observee and maximum number of open transaction to 10
66 (COAP_MAX_OPEN_TRANSACTIONS and COAP_MAX_OBSERVEES = 10)
